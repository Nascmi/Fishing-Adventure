# Repository Working Instructions

- Read the relevant files in `docs/` before changing the project.
- After every project change, update the appropriate documentation in `docs/` during the same task. At minimum, record player-facing work in `docs/CHANGELOG.md`; also update every design, system, roadmap, testing, balance, or workflow document affected by the change.
- Do not defer documentation cleanup to a later truth pass. Code and documentation must describe the same current behavior at handoff.
- Run the affected tests and production build before considering implementation complete.
- Do not create a Git commit unless the user explicitly requests one.
